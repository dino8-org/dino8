BUILD A BREAKOUT GAME

Build a classic brick-breaking game from scratch in 10 steps.

STEP 1 Project setup

Create a new project and configure it for our 128×128 pixel game.

$ dino8 create breakout --empty
$ cd breakout

Open conf.lua and set the display size. The scale sets the window multiplier so each pixel is drawn 4×4 on screen.

We also define our sound effects up front in sfx_clips. Each clip has a path to a wav file and a pitch range — the engine picks a random pitch between the two values each time it plays, so repeated sounds don't feel robotic.

Grab the tutorial sound effects with the get command. Run this from inside your project folder — it downloads and extracts the wav files into assets/audio/ automatically:

$ dino8 get tutorial breakout

Open main.d8 — this is your game entry point. Clear it out — we'll build everything from scratch.

CONF.LUA 
return {
  title = "Breakout",
  version = "0.1.0",
  author = "Dino Dev",
  engine = 0,
  width = 128,
  height = 128,
  scale = 4,
  entry = "main.d8",
  sfx_clips = {
    thud = {
      path  = "assets/audio/thud.wav",
      pitch = { 0.8, 1.2 },
    },
    hit = {
      path  = "assets/audio/break.wav",
      pitch = { 0.9, 1.3 },
    },
  },
}
STEP 2 Draw the paddle

Start with the simplest visible thing: a white rounded rectangle near the bottom of the screen.

New functions:
_draw() a callback the engine calls every frame to render
cls(c) clear the screen with color c — a 1-based index into the palette
rrectf(x0,y0,x1,y1,r,c) filled rounded rectangle

Run it with dino8 breakout/ and you should see a small white bar at the bottom.

MAIN.D8 
local paddle_x = 52
local paddle_y = 120
local paddle_w = 24
local paddle_h = 3

function _draw()
  cls(1)
  rrectf(paddle_x, paddle_y, paddle_x + paddle_w, paddle_y + paddle_h, 1, 7)
end
STEP 3 Move the paddle

Add an _update() function that runs every frame before drawing. Check for left/right input and move the paddle.

New functions:
_update() called every frame for logic
btn(name) true while a button is held
DT seconds since last frame (~1/60)
max(a,b) / min(a,b) clamp values

We multiply speed by DT so movement is framerate-independent. The min and max calls keep the paddle on screen.

MAIN.D8 
local paddle_speed = 90

function _update()
  if btn("left") then
    paddle_x = max(0, paddle_x - paddle_speed * DT)
  end
  if btn("right") then
    paddle_x = min(128 - paddle_w, paddle_x + paddle_speed * DT)
  end
end
STEP 4 Add the ball

Draw a small circle and give it a velocity (dx, dy). Each frame, update its position and bounce off the walls.

New functions:
circf(x,y,r,c) filled circle
sfx(clip) play a sound effect

We also play the thud sound effect we configured in conf.lua. The sfx_clips table is a global — use sfx(sfx_clips.thud) to play a clip.

The ball moves diagonally upward. It bounces off the left, right, and top walls but passes through the bottom — we'll handle that in step 6.

MAIN.D8 
local ball_x = 64
local ball_y = 100
local ball_r = 2
local ball_dx = 50
local ball_dy = -70

function _update()
  -- input ...

  -- move ball
  ball_x = ball_x + ball_dx * DT
  ball_y = ball_y + ball_dy * DT

  -- wall bounces
  if ball_x - ball_r < 0 then
    ball_x = ball_r
    ball_dx = -ball_dx
    sfx(sfx_clips.thud)
  end
  if ball_x + ball_r > 128 then
    ball_x = 128 - ball_r
    ball_dx = -ball_dx
    sfx(sfx_clips.thud)
  end
  if ball_y - ball_r < 0 then
    ball_y = ball_r
    ball_dy = -ball_dy
    sfx(sfx_clips.thud)
  end
end

function _draw()
  -- ...
  circf(ball_x, ball_y, ball_r, 16)
end
STEP 5 Paddle collision

When the ball hits the paddle, bounce it upward. We also steer the ball based on where it hit — hitting the left edge sends it left, the right edge sends it right.

The hit value ranges from 0 (left edge) to 1 (right edge). Subtracting 0.5 gives us a range of -0.5 to 0.5, which we scale into horizontal speed. This gives the player control over the ball's direction.

MAIN.D8 
function _update()
  -- input ...
  -- move ball ...
  -- wall bounces ...

  -- paddle collision
  if ball_dy > 0 and
     ball_y + ball_r >= paddle_y and
     ball_y - ball_r <= paddle_y + paddle_h and
     ball_x >= paddle_x and
     ball_x <= paddle_x + paddle_w then
    ball_dy = -ball_dy
    ball_y = paddle_y - ball_r
    local hit = (ball_x - paddle_x) / paddle_w
    ball_dx = (hit - 0.5) * 140
    sfx(sfx_clips.thud)
  end
end
STEP 6 Ball reset

When the ball falls past the bottom of the screen, reset it to the center. In the full game you'd lose a life here, but for now we just put it back.

MAIN.D8 
function _update()
  -- input ...
  -- move ball ...
  -- wall bounces ...
  -- paddle collision ...

  -- ball fell off bottom
  if ball_y > 132 then
    ball_x = 64
    ball_y = 100
    ball_dx = 50
    ball_dy = -70
  end
end
STEP 7 Draw bricks

Create a grid of colored bricks at the top of the screen. We use _init() to build the brick table once at startup.

New functions:
_init() called once at startup
add(t, v) append value to table
all(t) iterate over a table

Each brick is a table with position, size, color, and an alive flag. We lay out 8 columns and 5 rows, each row a different color.

MAIN.D8 
local bricks = {}
local brick_w = 14
local brick_h = 6
local brick_colors = {8, 9, 10, 11, 12}

function _init()
  for row = 0, 4 do
    for col = 0, 7 do
      add(bricks, {
        x = 4 + col * 15,
        y = 16 + row * 7,
        w = brick_w,
        h = brick_h,
        color = brick_colors[row + 1],
        alive = true,
      })
    end
  end
end

function _draw()
  cls(1)
  for b in all(bricks) do
    if b.alive then
      rectf(b.x, b.y, b.x + b.w - 1, b.y + b.h - 1, b.color)
    end
  end
  -- paddle ...
  -- ball ...
end
STEP 8 Brick collision

Check if the ball overlaps any alive brick. If it does, destroy the brick and bounce the ball.

The collision check is the same AABB approach we used for the paddle — check if the ball's bounding circle overlaps the brick's rectangle.

MAIN.D8 
function _update()
  -- input ...
  -- move ball ...
  -- wall bounces ...
  -- paddle collision ...

  -- brick collisions
  for b in all(bricks) do
    if b.alive and
       ball_x + ball_r > b.x and
       ball_x - ball_r < b.x + b.w and
       ball_y + ball_r > b.y and
       ball_y - ball_r < b.y + b.h then
      b.alive = false
      ball_dy = -ball_dy
      sfx(sfx_clips.hit)
    end
  end

  -- ball reset ...
end
STEP 9 Score

Add a score variable, increment it when a brick is destroyed, and display it using the _ui() callback. This works like _draw() but isn't affected by camera transforms — perfect for HUD elements.

New functions:
_ui() called every frame for HUD overlay
print(text,x,y,c) draw text at position
MAIN.D8 
local score = 0

function _update()
  -- ...

  -- brick collisions
  for b in all(bricks) do
    if b.alive and ... then
      -- ...
      score = score + 10
    end
  end

  -- ...
end

function _ui()
  print(score, 2, 2, 7)
end
STEP 10 Polish

Add screen shake when bricks are hit for a satisfying crunch feel.

New functions:
cam_shake(intensity, duration) screen shake effect

Just add one line inside the brick collision block. The screen will jolt briefly on each hit.

That's it — you've built a complete breakout game! From here you can extend it with:

  • Lives and game over screen
  • Multiple levels with harder bricks
  • Power-ups (bigger paddle, multi-ball)

Check out the full example with dino8 examples to see all of these in action.

MAIN.D8

    end
  end

  -- ...
end

ALTERNATIVES

LOADING SOUNDS IN CODE

Instead of defining sounds in conf.lua, you can load them in code with load_sfx() and store the returned ID:

local snd_thud
local snd_hit

function _init()
  snd_thud = load_sfx("assets/audio/thud.wav")
  snd_hit  = load_sfx("assets/audio/break.wav")
end

-- then play with:
sfx(snd_thud)
sfx(snd_hit)
