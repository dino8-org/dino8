BUILD A PINBALL MACHINE

Plan a layout with debug tools, build it from data, then bring it to life with physics — flippers, bumpers, a spring launcher, and smooth camera follow.

STEP 1 Grid & debug view

Create a new project. We'll use a 160×260 screen so you can see the full 128×256 playfield without scrolling — this makes it easy to plan where everything goes. We'll shrink to 128×128 at the end when we add camera follow.

$ dino8 create pinball --empty
$ cd pinball

Draw an 8-pixel grid over the playfield and show the mouse position snapped to the nearest 4 pixels. Press B to toggle the debug overlay on and off.

New functions:
cls(color) clear the screen
rect(x1, y1, x2, y2, color) draw a rectangle outline
pset(x, y, color) draw a single pixel
print(text, x, y, color) draw text
mouse_screen_pos() returns mouse x, y in screen coordinates
btnp(name) true on the frame a button is first pressed

return {

  engine = 0,
  width = 152,
  height = 260,
  scale = 3,

main.d8
local W, H = 128, 256
local SW, SH = 152, 260
local GRID = 8
local SNAP = 4
local debug_on = true

function _update()
  if btnp("action_b") then
    debug_on = not debug_on
  end
end

function draw_grid()
  for gx = 0, SW / GRID - 1 do
    for gy = 0, SH / GRID - 1 do
      rect(gx * GRID, gy * GRID,
           gx * GRID + GRID,
           gy * GRID + GRID, 2)
    end
  end
  -- screen boundary at x=128
  line(W, 0, W, SH, 3)
end

function draw_cursor()
  local mx, my = mouse_screen_pos()
  local sx = flr(mx / SNAP + 0.5) * SNAP
  local sy = flr(my / SNAP + 0.5) * SNAP
  pset(sx, sy, 7)
  print(sx .. "," .. sy, 2, 252, 7)
end

function _draw()
  cls(1)
  if debug_on then
    draw_grid()
    draw_cursor()
  end
end
STEP 2 Measuring tool

Before we place any walls or bumpers, we need a way to measure distances on the playfield. We'll build a drag-to-measure tool with three modes:

  • Line — click and drag to draw a line. Shows start and end coordinates.
  • Rect — click and drag from the top-left corner. Shows x, y, width, and height.
  • Circ — click the center and drag outward. Shows center coordinates and radius.

Press A to cycle between modes. Measurements persist on screen so you can place several at once. Right-click to clear them all.

New functions:
mouse_btn(n) true while mouse button n is held (1 = left, 2 = right)
line(x1, y1, x2, y2, color) draw a line
circ(x, y, r, color) draw a circle outline
sqrt(n) square root
main.d8
-- measuring tool
local modes = { "line", "rect", "circ" }
local mode_idx = 1
local measures = {}
local dragging = false
local drag_x, drag_y = 0, 0

function snap(v)
  return flr(v / SNAP + 0.5) * SNAP
end

function update_measure()
  if not debug_on then return end

  if btnp("action_a") then
    mode_idx = mode_idx % #modes + 1
  end

  local mx, my = mouse_screen_pos()
  local sx, sy = snap(mx), snap(my)

  if mouse_btn(2) then
    measures = {}
    return
  end

  if mouse_btn(1) then
    if not dragging then
      dragging = true
      drag_x, drag_y = sx, sy
    end
  elseif dragging then
    dragging = false
    add(measures, {
      mode = modes[mode_idx],
      x1 = drag_x, y1 = drag_y,
      x2 = sx, y2 = sy,
    })
  end
end

function draw_measure(m)
  if m.mode == "line" then
    line(m.x1, m.y1, m.x2, m.y2, 11)
    print(m.x1 .. "," .. m.y1,
          m.x1 + 2, m.y1 - 6, 11, 1)
    print(m.x2 .. "," .. m.y2,
          m.x2 + 2, m.y2 - 6, 11, 1)
  elseif m.mode == "rect" then
    local w = m.x2 - m.x1
    local h = m.y2 - m.y1
    rect(m.x1, m.y1, m.x2, m.y2, 11)
    print(m.x1 .. "," .. m.y1,
          m.x1 + 2, m.y1 - 10, 11, 1)
    print("w:" .. w .. " h:" .. h,
          m.x1 + 2, m.y2 + 2, 11, 1)
  elseif m.mode == "circ" then
    local dx = m.x2 - m.x1
    local dy = m.y2 - m.y1
    local r = flr(sqrt(dx*dx + dy*dy))
    circ(m.x1, m.y1, r, 11)
    print(m.x1 .. "," .. m.y1,
          m.x1 + 2, m.y1 - 6, 11, 1)
    print("r:" .. r,
          m.x1 + r + 2, m.y1, 11, 1)
  end
end

function draw_measures()
  if not debug_on then return end

  for _, m in ipairs(measures) do
    draw_measure(m)
  end

  -- live preview while dragging
  if dragging then
    local mx, my = mouse_screen_pos()
    draw_measure({
      mode = modes[mode_idx],
      x1 = drag_x, y1 = drag_y,
      x2 = snap(mx), y2 = snap(my),
    })
  end

  print("mode: " .. modes[mode_idx],
        2, 2, 11, 1)
end

STEP 3 Wall lines

Now we'll create a separate data file for the layout. This keeps all coordinates in one place and separates data from game logic. Use the measuring tool to find positions and lengths, then write them into layout.lua.

We load it with read_file() and load() — the file returns a Lua table, so we get structured data back. For now it only has wall lines.

Draw the lines in _draw() by looping through the data. No physics yet — these are just visual guides.

TIP

Because the layout is a plain data file, you could build a tool that generates it automatically — for example, a visual level editor that writes out the coordinates.

New functions:
read_file(path) read a file as a string
load(str)() compile and run a Lua string — returns whatever the string returns

STEP 4 Wall collision shapes

Now we turn the visual lines into physics. For each line in the layout data, create an edge body — a line segment that things can bounce off.

Call physics_draw() at the end of _draw() to overlay the physics shapes in a different colour. If the outlines match your wall lines, the data is correct.

New functions:
body(x, y, config) create a physics body — config sets shape, kind, bounce, friction, etc.
physics_draw(color) draw all physics bodies as wireframes — great for debugging

Edge bodies use shape = "edge". The start point is (x, y) and the end point is (x2, y2). Setting kind = "static" means they never move.

main.d8
local walls = {}

function create_walls()
  for _, ln in ipairs(layout.lines) do
    add(walls, body(ln[1], ln[2], {
      shape = "edge",
      kind = "static",
      x2 = ln[3], y2 = ln[4],
      friction = 0.3,
      bounce = 0.4,
    }))
  end
end

function _init()
  create_walls()
end

-- at the end of _draw:
if debug_on then
  physics_draw(8)
end
STEP 5 Entity placement

Add bumpers, pegs, and flipper pivot positions to layout.lua. Use the circ mode of the measuring tool to find centre coordinates and radii.

We draw them visually so we can see the layout, but we don't create physics bodies yet — those come later when we implement each mechanic. For now it's just data and drawing.

Flipper pivots are marked with small crosses. They don't need a radius since flippers are rectangles.

STEP 6 Launcher

Time for physics! The ball starts as a static body sitting on the launcher. Hold down to pull the launcher back — the ball follows it down. Release to fire.

On release, the static ball is destroyed and recreated as a dynamic body with an upward body_impulse. A simple state machine tracks ready → charging → playing.

We call physics_step(DT) every frame to advance the simulation — without this, nothing moves.

New functions:
gravity(gx, gy) set global gravity vector
physics_step(dt) advance the simulation — must be called every frame
body(x, y, opts) create a physics body
body_pos(id) get body position — returns x, y
body_set_pos(id, x, y) move a body to a position
body_impulse(id, ix, iy) apply an instant force
body_destroy(id) remove a body from the world
btn(name) true while a button is held
main.d8
local ball
local ball_r = 3
local state = "ready"
local launch_charge = 0
local launch_max = 1.0
local launch_power = 400
local launch_pull = 15

function pull_y()
  return (launch_charge / launch_max) * launch_pull
end

function spawn_ball()
  if ball then body_destroy(ball) end
  local la = layout.launcher
  ball = body(la.x, la.y, {
    shape = "circ", kind = "static",
    r = ball_r, bounce = 0.4,
    friction = 0.3,
  })
  state = "ready"
  launch_charge = 0
end

function launch_ball(power)
  local la = layout.launcher
  body_destroy(ball)
  ball = body(la.x, la.y, {
    shape = "circ", kind = "dynamic",
    r = ball_r, bounce = 0.4,
    friction = 0.3, mass = 1,
  })
  body_impulse(ball, 0, -power)
end

function _init()
  gravity(0, 100)
  create_walls()
  spawn_ball()
end

function _update()
  -- debug toggle + measure tool ...

  physics_step(DT)
  if not ball then return end

  local la = layout.launcher

  if state == "ready" then
    if btn("down") then
      launch_charge = 0
      state = "charging"
    end
  elseif state == "charging" then
    if btn("down") then
      launch_charge = min(
        launch_charge + DT, launch_max)
      body_set_pos(ball,
        la.x, la.y + pull_y())
    else
      local power = (launch_charge
        / launch_max) * launch_power
      launch_ball(power)
      state = "playing"
    end
  elseif state == "playing" then
    local _, by = body_pos(ball)
    if by > H + 10 then
      spawn_ball()
    end
  end
end

function _draw()
  cls(1)
  draw_walls()
  draw_entities()

  -- launcher
  local la = layout.launcher
  local top = la.y + pull_y() + ball_r + 2
  rectf(130, 64, 142, H, 2)
  rectf(132, top, 140, H - 2, 10)

  -- ball
  if ball then
    local bx, by = body_pos(ball)
    circf(bx, by, ball_r, 16)
    circ(bx, by, ball_r, 0)
  end

  -- debug ...
end

STEP 8 Bumpers & collisions

Now we create physics bodies for the bumpers and pegs using the positions from layout.lua.

Bumpers have bounce = 1.5 — above 1.0 means the ball leaves faster than it arrived. Classic pinball energy boost.

We define a collision callback _on_collide to detect hits, award points, and shake the screen. Instead of tracking timers manually, we use the tween system to animate the hit effect.

tween() animates properties on a table over time. chain() sequences tweens so they play one after another — here we expand the bumper border outward, then spring it back with an elastic ease. The on_complete callback resets the colour when the animation finishes.

New functions:
_on_collide(a, b) magic function called when two bodies collide — receives both body IDs
cam_shake(intensity, duration) screen shake effect
tween(target, duration, props, ease) animate properties on a table over time
chain(...) play tweens in sequence — each starts when the previous finishes
tween_cancel(handle) stop a running tween
Ease easing curves — Ease.Quad, Ease.Elastic, Ease.Bounce, etc.
main.d8
local bumpers = {}
local pegs = {}

function create_bumpers()
  for _, b in ipairs(layout.bumpers) do
    local id = body(b.x, b.y, {
      shape = "circ",
      kind = "static",
      r = b.r, bounce = 1.5,
      friction = 0.1,
    })
    add(bumpers, {
      id = id, x = b.x, y = b.y,
      r = b.r,
      col = 10,
      border_r = b.r - 1,
      tw = nil,
    })
  end

  for _, p in ipairs(layout.pegs) do
    local id = body(p.x, p.y, {
      shape = "circ",
      kind = "static",
      r = p.r, bounce = 0.4,
      friction = 0.2,
    })
    add(pegs, {
      id = id, x = p.x, y = p.y,
      r = p.r,
    })
  end
end

function bumper_hit(bmp)
  -- cancel any running animation
  if bmp.tw then
    tween_cancel(bmp.tw)
  end

  bmp.col = 7 -- flash white
  bmp.border_r = bmp.r - 1

  -- expand border out, then spring back
  bmp.tw = chain(
    tween(bmp, 0.1,
      { border_r = bmp.r + 3 },
      Ease.Quad),
    tween(bmp, 0.3,
      { border_r = bmp.r - 1 },
      Ease.Elastic),
    function()
      bmp.col = 10
      bmp.tw = nil
    end
  )
end

function _on_collide(a_id, b_id)
  for _, bmp in ipairs(bumpers) do
    if a_id == bmp.id
      or b_id == bmp.id
    then
      bumper_hit(bmp)
      score = score + 100
      cam_shake(2, 0.2)
    end
  end
end

STEP 9 Flippers

Flippers are kinematic bodies. Unlike dynamic bodies, they don't respond to forces or gravity — you control them entirely with code. But they do push dynamic bodies (the ball) when they move.

We define the flipper shape as a polygon — a trapezoid that's fat at the pivot and thin at the tip. The points start at x=−1 (the pivot end) and extend out to flip_len, so the body rotates from the edge, not the centre.

Press left arrow for the left flipper, right arrow for the right. We set body_set_angular_vel each frame so the physics engine moves them, then read back the actual angle with body_angle and clamp it to the allowed range.

New functions:
body(x, y, { shape = "poly", points = {} }) polygon body from a flat list of x,y pairs
body_set_angular_vel(id, vel) set rotational speed (radians/sec)
body_angle(id) read back the current angle after physics step
mid(lo, val, hi) clamp a value between lo and hi
polyf(pts, col), poly(pts, col) draw filled/outlined polygon

Run the game and try the flippers. If the positions don't feel right, go back to layout.lua and adjust the flipper x/y values — use the measuring tool to find coordinates that sit nicely above the drain walls.

main.d8
local flip_l, flip_r
local flip_len = 36
local flip_fat = 5
local flip_thin = 2
local flip_rest = 0.35
local flip_active = -0.6
local flip_up_speed = 12
local flip_down_speed = 4
local flip_l_angle = 0
local flip_r_angle = 0

-- tapered shape: fat at pivot, thin at tip
local l_pts = {
  -1, -flip_fat,
  flip_len, -flip_thin,
  flip_len,  flip_thin,
  -1,  flip_fat,
}
local r_pts = {
  1, -flip_fat,
  -flip_len, -flip_thin,
  -flip_len,  flip_thin,
  1,  flip_fat,
}

function create_flippers()
  local fl = layout.flippers.left
  local fr = layout.flippers.right
  flip_l = body(fl.x, fl.y, {
    shape = "poly",
    kind = "kinematic",
    points = l_pts,
    friction = 0.8, bounce = 0.3,
  })
  flip_l_angle = flip_rest

  flip_r = body(fr.x, fr.y, {
    shape = "poly",
    kind = "kinematic",
    points = r_pts,
    friction = 0.8, bounce = 0.3,
  })
  flip_r_angle = -flip_rest
end

-- add to _init:
-- create_flippers()

-- in _update, before physics_step:
local target_l = flip_rest
local target_r = -flip_rest
if btn("left") then
  target_l = flip_active
end
if btn("right") then
  target_r = -flip_active
end

local diff_l = target_l - flip_l_angle
local diff_r = target_r - flip_r_angle
local spd_l = (diff_l < 0)
  and flip_up_speed or flip_down_speed
local spd_r = (diff_r > 0)
  and flip_up_speed or flip_down_speed
body_set_angular_vel(flip_l, diff_l * spd_l)
body_set_angular_vel(flip_r, diff_r * spd_r)

-- after physics_step, read back and clamp:
flip_l_angle = body_angle(flip_l)
flip_r_angle = body_angle(flip_r)
flip_l_angle = mid(
  flip_active, flip_l_angle, flip_rest)
flip_r_angle = mid(
  -flip_rest, flip_r_angle, -flip_active)
body_set_angle(flip_l, flip_l_angle)
body_set_angle(flip_r, flip_r_angle)

-- drawing a rotated flipper:
function draw_flipper(fx, fy, pts, angle)
  local c, s = cos(angle), sin(angle)
  local rotated = {}
  for i = 1, #pts, 2 do
    local lx, ly = pts[i], pts[i + 1]
    add(rotated, fx + lx * c - ly * s)
    add(rotated, fy + lx * s + ly * c)
  end
  polyf(rotated, 9)
  poly(rotated, 0)
end

-- in _draw:
local fl = layout.flippers.left
local fr = layout.flippers.right
draw_flipper(fl.x, fl.y, l_pts, flip_l_angle)
draw_flipper(fr.x, fr.y, r_pts, flip_r_angle)
STEP 10 UI

Add a HUD with score and remaining balls. The _ui() callback draws after _draw() and isn't affected by the camera — perfect for score displays. This matters in the next step when we add camera follow.

We also show a blinking "hold down to launch" hint when the ball is ready.

STEP 11 Camera follow

The playfield is 256 pixels tall but we only want to show 128 at a time. Change the resolution back to 128×128 and add camera follow so the view tracks the ball.

cam_follow smoothly moves the camera toward the ball. The time parameter controls smoothness — 0.15 gives a responsive but buttery follow.

cam_bounds prevents the camera from showing empty space beyond the playfield edges. And cam_offset with a velocity-based offset gives the player a preview of where the ball is heading.

New functions:
cam_follow(x, y, time) smoothly move camera toward a point
cam_bounds(x1, y1, x2, y2) clamp camera within bounds
cam_offset(ox, oy) offset from the follow target
body_vel(id) get velocity — returns vx, vy

Note that _ui() still draws in screen space — the score and ball count stay fixed in the corners regardless of where the camera is.

conf.d8
return {

  author = "Dino Dev",
  engine = 0,
  width = 128,
  height = 128,
  scale = 3,

}
main.d8
local look_ahead = 0

-- in _init:
cam_bounds(0, 0, W + GRID * 6, H)

-- in _update, after physics_step:
if ball then
  local bx, by = body_pos(ball)
  local _, vy = body_vel(ball)

  -- look ahead based on velocity
  local target = (vy / 200) * 32
  target = mid(-32, target, 32)
  look_ahead = look_ahead
    + (target - look_ahead)
    * min(1, 8 * DT)

  cam_follow(bx, by, 0.15)
  cam_offset(0, look_ahead)
end
